📁 HTML 파일 받기

이 도구는 단일 HTML 파일(거의다되는_자산관리_v2.html) 하나로 동작합니다. 별도 설치나 서버가 필요 없습니다.

1
파일을 전달받는다 — 이메일, 카카오톡, USB, 구글 드라이브 등 어떤 방법으로든 .html 파일을 받습니다.
2
원하는 폴더에 저장 — 바탕화면, 내 문서 등 어디든 상관없습니다.
3
크롬 브라우저로 열기 — 파일을 더블 클릭하거나 크롬에 드래그 앤 드롭합니다.
ℹ️ 기본 포트폴리오 입력 및 수동 자산 기록은 인터넷 없이도 가능합니다. 단, 실시간 종가 자동 조회 및 매매일지 자동 계산 기능을 사용하려면 인터넷 + CORS 확장 프로그램 + API 키가 필요합니다.

초기 설정 — CORS 확장 프로그램 & API 키 발급
ℹ️ 왜 필요한가요?
이 프로그램은 별도 서버 없이 개인 브라우저에서 모든 계산과 조회를 처리합니다. 그런데 브라우저 보안 정책(CORS)상, 한국거래소 등 외부 사이트 데이터를 직접 가져오는 것이 기본적으로 차단됩니다.

따라서 실시간 종가 조회 및 매매일지 자동 계산을 사용하려면 아래 두 가지가 반드시 필요합니다:
① CORS 우회 확장 프로그램 (브라우저 보안 임시 해제)
② 공공데이터포털 API 키 (공식 주식·ETF 데이터 호출권)
1️⃣ 1단계: CORS 확장 프로그램 설치
1
크롬 웹스토어에 접속합니다.
chrome.google.com/webstore
2
「Allow CORS: Access-Control-Allow-Origin」을 검색하여 브라우저에 추가합니다.
3
설치 후 브라우저 우측 상단 퍼즐 조각 🧩 아이콘을 눌러 해당 확장을 찾습니다.
4
큰 전원 버튼을 눌러 ON (아이콘이 컬러로 변함) 상태로 만들어 줍니다.
🚨 보안 주의 — 이 확장이 켜져 있으면 브라우저 보안이 낮아진 상태입니다. 자산관리 툴을 다 쓰고 나면 반드시 다시 OFF로 돌려두세요. 특히 인터넷 뱅킹·결제 시에는 꼭 끄고 사용하세요.
🛡️ 가장 안전한 방법 — 수동 ON/OFF 토글

로컬 파일(file:///C:/...)은 CORS 확장 화이트리스트 등록이 불가능합니다. 결론적으로 수동 ON/OFF 방식이 유일하고 가장 안전한 방법입니다:
  1. 평소 (은행 업무, 웹서핑) → 확장 프로그램 아이콘 회색 (OFF) 유지
  2. 자산관리 할 때 → HTML 파일을 열고, 데이터 업데이트 버튼 누르기 직전에 주황색 (ON)
  3. 완료 후 → 다시 클릭 → 회색 (OFF)
2️⃣ 2단계: 공공데이터포털 API 키 발급받기 (무료)
1
공공데이터포털 (data.go.kr)에 접속하여 회원가입 및 로그인합니다.
2
검색창에 아래 두 가지를 각각 검색하여 「활용신청」 버튼을 클릭합니다.

🔎 금융위원회_주식시세정보 — 일반 국내 주식 종가 조회용
🔎 금융위원회_증권상품시세정보 — ETF 종가 조회용
3
마이페이지 → 오픈API → 개발계정으로 이동합니다.
4
신청한 API를 클릭하면 일반 인증키 (Encoding) 문자열이 보입니다. 이 긴 영어+숫자 조합의 키를 복사합니다.
🚨 동기화 시간 1~2시간 소요 — 발급 직후 바로 사용하면 "키 오류"가 날 수 있으니, 1~2시간 후에 테스트하세요.
🚨 키 유출 금지 — API 키는 본인만의 고유 비밀번호입니다. 키가 입력된 화면을 캡처해 블로그에 올리거나 GitHub에 공개하면 타인이 사용해 호출 한도 초과(429 에러)가 발생할 수 있습니다.

🔑 API 키 입력 위치

1
HTML 파일을 크롬으로 엽니다. (CORS 확장 프로그램이 ON 상태여야 합니다.)
2
상단 메뉴에서 「2) 자산 추이 분석」 탭을 클릭합니다.
3
하단의 「📋 매매일지 → 자산추이 자동 계산」 섹션을 클릭해 펼칩니다.
4
복사해 둔 API 키를 각 칸에 붙여넣습니다:

• 국내주식 종가 조회용: 「금융위원회_주식시세정보」 키
• ETF 종가 조회용: 「금융위원회_증권상품시세정보」 키
(두 서비스에서 발급된 키가 동일하다면 같은 키를 양쪽에 입력하면 됩니다.)
5
옆의 「저장」 버튼을 클릭합니다. 이후 자동으로 저장되어 다음에 열 때도 유지됩니다.
✅ 종목명 띄어쓰기 엄수 — ETF 매매일지 입력 시 TIGER K방산우주처럼 공식 명칭의 띄어쓰기를 정확히 맞춰야 API가 올바르게 종목을 찾아옵니다.

🏦 계좌 / 그룹 관리

종목을 계좌별·그룹별로 분류 관리합니다. 상단 「🏦 계좌 / 그룹 관리」 카드를 클릭해 펼칩니다.

  • 각 계좌 안에서 그룹 추가/삭제 (ETF, 개별주, 채권 등)
  • 종목 행의 ☰ 핸들을 드래그해 다른 그룹으로 이동 가능
💱 USD 매도 우대율 (계좌별)

각 계좌 카드 안에 💱 USD 매도 우대율 [___] % 입력 필드가 있습니다.
기본값은 100% (현재 환율 그대로 적용). 예를 들어 99.7을 입력하면 현재 환율의 99.7%로 계산됩니다.

값을 변경하면:
• 오른쪽에 → 실제 적용환율(원)이 즉시 표시됩니다 (예: → 1,382.4원)
• 잠깐 「✓ 적용됨」 초록색 문구가 표시되며, 포트폴리오 손익이 즉시 재계산됩니다.
• 계좌를 삭제하거나 이름을 변경하면 우대율도 함께 갱신/삭제됩니다.

원인설명
NXT(넥스트레이드) 종가 넥스트레이드는 KRX와 별도 거래소로 종가가 다를 수 있습니다. 앱은 KRX 정규장 종가만 지원하며 NXT 종가는 반영되지 않습니다.
시간외 단일가 정규장 종가(15:30)와 시간외 단일가(15:30~16:00) 사이의 차이입니다. HTS가 시간외 단일가를 표시 중이라면 앱의 종가와 다를 수 있습니다.
수정주가 적용 여부 배당락·액면분할 등 이후 수정주가를 적용하는 경우 HTS와 다를 수 있습니다. 네이버 API는 수정주가 기준을 따릅니다.
미국 주식 환율·시장 기준 Pre/After Market 가격이 정규장 종가와 다를 수 있고, Yahoo Finance의 regularMarketPrice 기준을 사용합니다.
확인 방법: 네이버 금융(finance.naver.com)에서 종목을 검색해 나오는 종가와 앱의 값을 비교하세요. 네이버 금융 종가와 앱이 일치한다면 앱은 정상 동작 중입니다.

🇺🇸 미국 주식 손익 — 원화 / 달러 이중 표시 핵심

종목코드가 영숫자 6자리가 아닌 경우 (예: AAPL, TSLA) 미국 주식으로 자동 인식합니다.

표시 항목계산 방식
원화 손익 (상단 큰 숫자) (현재가USD × 매도환율) − (평단가USD × 매수환율) × 수량
매도환율 = 현재 USD/KRW × 계좌 우대율(%)
달러 손익 (하단 작은 숫자) (현재가USD − 평단가USD) × 수량
💡 매수 시 환율은 종목 추가 시 입력한 매수 환율(avgFxRate)을 사용합니다. 계좌 관리에서 우대율을 설정하면 매도 시 환율이 자동 적용됩니다.

자주 묻는 질문 (FAQ)

❓ 자동 계산을 눌렀는데 오류가 납니다

  • CORS 확장 프로그램이 ON 상태인지 확인 (아이콘이 컬러)
  • API 키가 올바르게 입력·저장되어 있는지 확인
  • API 키 발급 직후라면 1~2시간 후에 재시도하세요
  • 인터넷 연결 상태 확인

❓ 429 에러가 뜹니다 (Too Many Requests)

공공데이터포털 API 호출 한도를 초과했거나, API 키가 외부에 노출되어 타인이 사용 중일 가능성이 있습니다.

  • 잠시(5~10분) 기다린 후 재시도
  • API 키를 외부에 공유한 적이 있다면 공공데이터포털에서 키를 재발급받으세요

❓ 브라우저를 껐다 켜면 데이터가 사라지나요?

아니요. 브라우저 자동저장으로 같은 컴퓨터·브라우저에서 파일을 다시 열면 자동 복원됩니다.

⚠️ 다른 기기·다른 브라우저·캐시 삭제 시에는 사라질 수 있습니다. 중요한 데이터는 CSV 백업(탭 3)도 주기적으로 받아두세요.

❓ 미국 주식을 입력하면 원화와 달러 손익이 다 나오나요?

네. 종목코드가 영숫자 6자리가 아닌 경우 (예: AAPL) 미국 주식으로 자동 인식되어 포트폴리오 상세 손익란에 원화 손익(큰 숫자)과 달러 손익(작은 글씨)이 함께 표시됩니다.

✅ 계좌 관리에서 계좌별 USD 매도 우대율(%)을 설정하면 매도 시 적용환율이 자동 반영됩니다. 값 변경 즉시 옆에 실제 적용 환율이 표시되며 「✓ 적용됨」 문구가 잠깐 나타납니다.

❓ 원금 관리 / 배당 관리에서 특정 행만 지우려면?

각 행 왼쪽의 체크박스를 클릭해 선택한 뒤, 「선택 삭제」 버튼을 누르면 선택한 행만 삭제됩니다. 헤더 체크박스로 전체 선택/해제도 가능합니다.

❓ 계좌를 여러 개 사용하려면?

자산 추이 분석 탭 → 「계좌 추가」로 계좌를 추가합니다. 각 계좌에 독립적으로 데이터를 입력하고, 하단 버튼에서 여러 계좌를 선택하면 합산 그래프를 볼 수 있습니다.

❓ ETF와 일반 주식 API 키가 다른가요?

공공데이터포털에서 발급받는 인증키(Encoding) 문자열은 두 서비스 모두 동일합니다. 같은 키를 주식용·ETF용 두 칸에 모두 입력해도 됩니다.

❓ 매매일지 목록에서 수정하면 그래프도 바뀌나요?

매매일지 목록의 수정/삭제는 매매 내역 기록만 변경합니다. 그래프에 반영하려면 자동 계산을 다시 실행하세요.

❓ 기간 총수익금은 플러스(+)인데, 기간 평가손익이 마이너스(-)로 나와요!

오류가 아니라 '수익 실현(익절)'을 아주 잘하셨다는 증거입니다!

평가손익 상태이던 주식을 매도해 수익을 확정 지으면, 그 금액이 「실현손익」으로 넘어갑니다.
이때 평가손익에서는 해당 금액이 빠져나가 마이너스로 표시될 수 있습니다.

✅ 기간 총수익금 = 평가손익 + 실현손익 — 두 항목을 합친 값이 실제로 번 돈입니다.
  • 평가손익 −300만 + 실현손익 +500만 = 총수익금 +200만원 → 200만원을 번 것입니다.
  • 익절 후 남은 보유 주식의 평가손익이 내려가면 평가 항목이 마이너스가 될 수 있습니다.
